# Architecture

## Обзор системы
React SPA, работающее поверх локальной файловой системы пользователя через `URL.createObjectURL`. Состояние приложения хранится в оперативной памяти (Zustand), а персистентные данные о прогрессе и структуре курсов — в IndexedDB (Dexie).

## Директории
- `src/components/`:
  - `VideoPlayer.jsx` (обертка HTML5 video)
  - `Controls.jsx` (кастомные элементы управления)
  - `Sidebar.jsx` (навигация по урокам)
- `src/pages/`:
  - `Setup.jsx` (выбор директории курса)
  - `Player.jsx` (основной Layout плеера)
- `src/db/database.js`: Схема Dexie.
- `src/store/playerStore.js`: Zustand стор.

## Потоки данных (Data Flow)
1. **Инициализация**: Пользователь выбирает папку -> Читаем файлы -> `database.js` сохраняет курс и видео (без самих бинарников).
2. **Воспроизведение**: `playerStore.js` берет File object -> `URL.createObjectURL` -> передает в `<video src>`.
3. **Прогресс**: Каждые 5 секунд `VideoPlayer` вызывает метод из `database.js` для обновления `watchedSeconds` в IndexedDB.

## Ключевые решения
- **Почему Dexie?** Удобная обертка над IndexedDB для сложных запросов (поиск видео по курсу, апдейт прогресса).
- **Почему Zustand?** Легковесный стейт-менеджер, идеально подходит для хранения ссылок на объекты File, которые нельзя класть в Redux/Context без проблем с сериализацией.
